UX

Moteur de recherche & ISBN
• Saisie manuelle ISBN-13 ou ISBN-10

Champs du livre
• Données bibliographiques : titre, auteur(s), nationalité, éditeur, collection, ISBN, date de publication originale, date de l'édition, pages
• Classification : 11 thèmes et sous-thèmes contextuels, cases vides par défaut
• Bouton "Suggérer via IA" pour le thème/sous-thème (Claude Haiku)
• Statuts : À lire, En cours, Lu, Étude, Collection, Néant
• Priorité de lecture (À lire / En cours)
• Date de lecture : deux selects mois / année
• Note ★ (Lu / Étude)
• Fiche de lecture (tous statuts) — bouton "Générer via IA" (Claude Haiku) + bouton "Effacer"
• Prompt fiche adapté selon le thème : résumé de l'histoire (Littérature) ou idées centrales (autres)
• Fiche générée préfixée par #Générée automatiquement par IA
• État physique : Neuf / Très bon / Bon / Correct / Abîmé
• Livre de collection (checkbox) — détection automatique : Pléiade, Bouquins, Quarto, Classiques Garnier, édition ancienne…
• Commentaire libre
Configuration Notion
• Token, Database ID, proxy Cloudflare et clé API Anthropic stockés en localStorage (jamais dans le code)
• Proxy Cloudflare Worker pour contourner les restrictions CORS
• Accepte l'URL Notion complète pour le Database ID
• Vérification de la connexion et sync des propriétés au clic "Enregistrer"
Ajout Notion
• Envoi direct via API Notion — création d'une page dans la base configurée
• Sync automatique des propriétés avant chaque envoi — création des champs manquants, détection des conflits de type
• Couverture envoyée comme image de couverture de la page Notion
• Contrôle doublon sur ISBN — trois options : Ajouter quand même / Mettre à jour statut & lecture / Annuler
• "Mettre à jour statut & lecture" : met à jour statut, priorité, date de lecture, note et fiche sur la page existante
Interface
• Fichier HTML autonome, zéro dépendance externe, déployé sur GitHub Pages
• Mode clair / sombre automatique (préférences système)
• Panneau de configuration et changelog accessibles en bas de page
